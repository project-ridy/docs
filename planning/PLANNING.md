# Ridy — 프로젝트 기획서

## 1. 비전

"함께 타는 길" — 이동을 공유하고, 비용을 절감하고, 탄소를 줄이다.

## 2. 타겟 유저

| 페르소나 | 특성 |
|---|---|
| 직장인 A | 매일 강남 출퇴근, 주차비 부담 |
| 대학생 B | 학교까지 1시간, 대중교통 환승 피곤 |
| 차주 C | 혼자 운전하기 아까운 빈 좌석 |

## 3. 핵심 기능 (MVP)

1. **매칭** — 출발지/도착지/시간 기반 자동 매칭
2. **정기 카풀** — 요일/시간 설정, 고정 파트너
3. **채팅** — 매칭된 탑승자 간 실시간 메시지
4. **비용 분담** — 거리 기반 자동 정산
5. **평점/리뷰** — 안전과 신뢰 확보

## 4. 차별화 포인트

- 정기 카풀에 특화 (일회성이 아닌 루틴 기반)
- 기업 연계 B2B 모델 (사내 카풀 플랫폼)
- 친환경 임팩트 시각화 (절감 CO₂ 표시)

## 5. 수익 모델

- 거래 수수료 (정산 금액의 N%)
- 기업 B2B 구독
- 프리미엄 매칭 (우선 매칭, 상세 프로필)

## 6. 마일스톤

| Phase | 내용 | 기간 |
|---|---|---|
| 1 | 기획 완료, 와이어프레임 | ~2주 |
| 2 | MVP 개발 (매칭 + 채팅) | ~6주 |
| 3 | 정산 + 평점 시스템 | ~3주 |
| 4 | 클로즈드 베타 | ~2주 |
| 5 | 오픈 베타 | ~4주 |
