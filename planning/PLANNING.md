# Ridy — 프로젝트 기획서

## 1. 비전

"같은 동네에서 같은 직장으로" — 동네 기반, 회사 인증, 출근 전용 카풀.

Ridy는 **회사 이메일 도메인 기반 폐쇄형 동네→직장 카풀 플랫폼**이다. 사용자는 가입 코드를 입력한 뒤 회사 이메일 인증코드와 비밀번호로 가입하고, 같은 회사 이메일 도메인 구성원끼리만 출근 방향 카풀을 연결한다. MVP의 매칭 방향은 **동네 출발지에서 회사 근무지로 이동하는 출근 카풀**로 제한한다.

## 2. 타겟 유저

| 페르소나 | 특성 |
|---|---|
| 차주 사원 | 매일 자차 출퇴근, 빈 좌석 아까움, 유류비 부담 |
| 탑승 사원 | 같은 동네 또는 인근 지하철역에서 회사까지 쾌적한 출근 원함 |
| 도메인 운영자 | 가입 코드 배포, 참여자 관리, 사내 커뮤니티 활성화 |

## 3. 핵심 가치

1. **사내 신뢰** — 같은 회사 이메일 도메인 구성원끼리만 매칭
2. **가입 코드 + 회사 이메일 인증** — 가입 코드 입력 후 회사 이메일로 인증코드를 발송하고 비밀번호를 설정해 외부인 차단
3. **동네 기반 출근 카풀** — 차주가 지도에서 출발 위치를 등록하고 탑승자는 주변 제공자를 선택
4. **비용 분담** — 거리 기반 자동 정산, 유류비 절감
5. **친환경** — 개인·회사 도메인 단위 절감 CO₂ 시각화

## 4. 핵심 기능 (MVP)

1. **가입 코드 기반 온보딩** — 가입 코드 입력 → 회사 이메일 인증코드 → 비밀번호 설정 → 회사 도메인 자동 매핑
2. **지도 기반 카풀 탐색** — 탑승자는 내 동네/현재 위치 주변의 회사행 카풀 제공자를 지도에서 확인
3. **차주 출발 위치 등록** — 차주는 동네 출발 위치, 회사 근무지, 출발 시간, 좌석 수를 등록
4. **채팅** — 매칭된 사원 간 실시간 메시지
5. **비용 분담** — 거리 기반 자동 정산
6. **평점/리뷰** — 사내 평판 관리

### MVP 제약

- 허용 방향: **동네 출발지 → 회사 근무지**
- 제외 방향: 퇴근 카풀, 직장 외 목적지, 임의 목적지 검색
- 차주 출발 위치는 승인 전 정확한 집 주소를 노출하지 않고, 지도 marker와 표시용 동네명만 노출한다.
- 탑승자는 지도 marker 또는 목록 카드에서 카풀 제공자를 선택해 요청한다.
- 승인 후 실제 만남 위치 또는 상세 안내를 채팅/요청 상세에서 공유한다.

## 5. 차별화 포인트

| 기존 서비스 | Ridy |
|---|---|
| 일반인 대상 오픈 카풀 | 회사 이메일 도메인 기반 폐쇄 카풀 |
| 신원 미확인 탑승자 | 같은 회사 도메인 구성원만 (가입 코드 + 회사 이메일 검증) |
| 임의 출발/도착 검색 | 동네 기반 회사행 출근 카풀 특화 |
| 개인 간 거래 | 회사 도메인으로 검증된 안전한 환경 |

## 6. 수익 모델

- **거래 수수료** — 정산 금액의 5%를 탑승자 결제 금액에 포함
- **클로즈드 커뮤니티 운영** — 가입 코드 발급·도메인 검증·기본 통계로 초기 파일럿 운영
- **후속 B2B 확장** — 회사 단위 계약, 플랜별 수수료 부담, 후속 관리자 화면, ESG 리포트는 별도 문서화 전까지 구현하지 않는다

후속 B2B 기능 범위, 플랜 정책, 후속 관리자 화면, ESG 리포트, 구독 결제의 BLOCKED 조건은 [B2B 기능 기획](./B2B_PLAN.md)을 기준으로 한다.

## 7. 마일스톤

| Phase | 내용 | 기간 |
|---|---|---|
| 1 | 기획 완료, 와이어프레임 | ~2주 |
| 2 | MVP 개발 (가입 코드 + 회사 이메일 검증 + 매칭 + 채팅) | ~6주 |
| 3 | 정산 + 평점 시스템 | ~3주 |
| 4 | 클로즈드 베타 (1개 회사 도메인 파일럿) | ~2주 |
| 5 | 오픈 베타 (다중 회사 도메인 지원) | ~4주 |

Phase 4~5의 회사 계약·구독·ESG 범위는 후속 확장으로 [B2B 기능 기획](./B2B_PLAN.md)에 정의한다.
