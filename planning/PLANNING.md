# Ridy — 프로젝트 기획서

## 1. 비전

"함께 타는 출퇴근" — 같은 회사, 같은 방향, 함께 타는 길.

Ridy는 **기업 단위 폐쇄형 카풀 플랫폼**이다. 같은 회사 사원끼리만 출퇴근 방향이 맞으면 카풀할 수 있도록 연결한다.

## 2. 타겟 유저

| 페르소나 | 특성 |
|---|---|
| 차주 사원 | 매일 자차 출퇴근, 빈 좌석 아까움, 유류비 부담 |
| 탑승 사원 | 대중교통으로 1시간+, 쾌적한 통근 원함 |
| 기업 관리자 | 교통비 지원 부담, ESG 실적, 조직 문화 개선 |

## 3. 핵심 가치

1. **사내 신뢰** — 같은 회사 사람이니까 안심. 별도 신원 인증 불필요
2. **초대 코드 가입** — 기업 관리자가 발급한 코드로만 가입, 외부인 차단
3. **정기 카풀** — 요일/시간 설정, 매일 같은 동료와 출퇴근
4. **비용 분담** — 거리 기반 자동 정산, 유류비 절감
5. **친환경** — 절감 CO₂ 시각화, 기업 ESG 실적 연동

## 4. 핵심 기능 (MVP)

1. **초대 코드 가입** — 관리자 발급 코드로 사원 가입, 회사 자동 매핑
2. **매칭** — 출발지/도착지/시간 기반, **같은 회사 사원만** 검색
3. **정기 카풀** — 요일/시간 설정, 고정 파트너
4. **채팅** — 매칭된 사원 간 실시간 메시지
5. **비용 분담** — 거리 기반 자동 정산
6. **평점/리뷰** — 사내 평판 관리

## 5. 차별화 포인트

| 기존 서비스 | Ridy |
|---|---|
| 일반인 대상 오픈 카풀 | 기업 단위 폐쇄 카풀 |
| 신원 미확인 탑승자 | 같은 회사 사원만 (초대 코드 검증) |
| 일회성 위주 | 정기 카풀 특화 (루틴 기반) |
| 개인 간 거래 | 기업이 관리하는 안전한 환경 |

## 6. 수익 모델

- **기업 B2B 구독** — 회사별 월 구독료 (사원 수 기반)
- **거래 수수료** — 정산 금액의 5% (기업 부담 또는 사원 분담)
- **관리자 대시보드** — 이용 통계, ESG 리포트, 초대 코드 관리

세부 B2B 기능 범위, 플랜 정책, 관리자 대시보드, ESG 리포트, 구독 결제의 BLOCKED 조건은 [B2B 기능 기획](./B2B_PLAN.md)을 기준으로 한다.

## 7. 마일스톤

| Phase | 내용 | 기간 |
|---|---|---|
| 1 | 기획 완료, 와이어프레임 | ~2주 |
| 2 | MVP 개발 (초대 코드 + 매칭 + 채팅) | ~6주 |
| 3 | 정산 + 평점 시스템 | ~3주 |
| 4 | 클로즈드 베타 (1개 기업 파일럿) | ~2주 |
| 5 | 오픈 베타 (다중 기업 지원) | ~4주 |

Phase 4~5의 기업 관리자·구독·ESG 범위는 [B2B 기능 기획](./B2B_PLAN.md)에 정의한다.
